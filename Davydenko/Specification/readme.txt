Задание 1: написать спецификацию в виде сценариев пользователя.
Инварианты: Меню позиций для доставки, обозначение доступных позиций, программа выполняется.
Пользовательский сценарий 1. Формирование заказа для клиента.
Действующие лицо: Клиент.
Предусловие: приложение delivery.exe запущено и открыта вкладка «Меню» (рисунок 1).
Основной сценарий:
•	Выбираем желаемую позицию из предложенного списка доступных позиций, после чего нажимает на кнопку «В корзину».
•	Переходим во вкладку «Корзина», где видим свой заказ.
•	Вводим запрашиваемые данные о себе, которых нет в базе данных (рисунок 2).
•	Нажимаем кнопку «Оформить заказ»

Альтернативный сценарий:
•	Выбираем желаемую позицию из предложенного списка доступных позиций, после чего нажимает на кнопку «В корзину»
•	Переходим во вкладку «Корзина», где видим свой заказ.
•	Нажимаем в таблице на позицию.
•	Нажимаем на кнопку «Удалить», исключая её из списка.
•	Вводим запрашиваемые данные о себе, которых нет в базе данных.
•	Нажимаем кнопку «Оформить заказ»

Альтернативный сценарий:
•	Выбираем желаемую позицию из предложенного списка доступных позиций, после чего нажимает на кнопку «В корзину»
•	Переходим во вкладку «Корзина», где видим свой заказ.
•	Вводим запрашиваемые данные о себе, которые атоматически заполняются (рисунок 3)
•	Нажимаем кнопку «Оформить заказ»
Постусловие: выводится сообщение о создание заказа, и он добавляется в базу данных (рисунок 4).


Пользовательский сценарий 2. Завершение заказа.
Действующие лицо: Администратор/Оператор.
Предусловие: приложение adm_dlvr.exe запущено (рисунок 5).
Основной сценарий:
•	Выбираем необходимый заказ.
•	В поле статсуса заказа ставим галочку на пункту «Заказ доставлен».
•	Нажимаем кнопку сохранить.
Постусловие: заказ исчезает из таблицы заказов, а оператор становится свободным (рисунок 6).


Пользовательский сценарий 3. Назначение доставщика на заказ.
Действующие лицо: Администратор/Оператор.
Предусловие: приложение adm_dlvr.exe запущено.
Основной сценарий:
•	Выбираем необходимый заказ.
•	Выбираем необходимого доставщика из таблицы доставщиков(рисунок 7).
•	Нажимаем кнопку «Назначить».
Постусловие: в поле «Номер заказа» у выбранного доставщика появляется выбранный заказ, у предыдущего доставщика этот заказ исчезает (рисунок 8).


Пользовательский сценарий 4. Снятие доставщика с заказа.
Действующие лицо: Администратор/Оператор.
Предусловие: приложение adm_dlvr.exe запущено.
Основной сценарий:
•	Выбираем необходимого доставщика из списка (рисунок 9).
•	Нажимаем кнопку «Отменить».
Постусловие: в поле «Номер заказа» у выбранного доставщика исчезает заказ (рисунок 10).
