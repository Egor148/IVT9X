Спецификация в виде сценария использования
Инварианты: программа запущена, окно видимо.

Пользовательский сценарий 1: Авторизация в программе 

Действующие лицо: администратор, преподаватель, ученик.

Предусловие: отображается окно авторизации.

Основной сценарий:
•   Заполнить поле логина
•   Заполнить поле пароля 
•   Выбрать тип пользователя администратор
•   Нажать кнопку "Подлючиться" 

Постусловие: если пользователь с такими данными существует, откроется окно программы. 
Если логин или пароль неправильные, то отображается окно с соотвествующей надписью. 

Пользовательский сценарий 2: Просмотр информации об ученике
Действующее лицо: Администратор

Предусловие: пользователь авторизирован как администратор.

Основной сценарий:
•   Нажать на кнопку "Ученики" в левой панел
•   Выбрать нужного ученика из списка
•   Нажать на кнопку "Информация" в левой панели

Постусловие: открывается окно с полной информацией об ученике.

Для просмотра информации о преподавателе и записи в расписании используется аналогичный сценарий.

Пользовательский сценарий 4: Добавление нового ученика
Действующее лицо: Администратор

Предусловие: пользователь авторизирован как администратор.

Основной сценарий:
•   Нажать на кнопку "Ученики" в левой панели
•   Нажать на кнопку "Добавить" в левой панели
•   Ввести ID
•   Ввести Фамилию
•   Ввести Имя 
•   Ввести Отчество
•   Ввести Где учится
•   Ввести В какие дни учится
•   Ввести Продолжительность занятий
•   Ввести Предпочительное время занятий
•   Ввести Пароль
•   Нажать кнопку "Добавить ученика"

Постусловие: в базе данных появится новый пользователь ученик, окно добавление закроется.
