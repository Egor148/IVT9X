Задание 1: написать спецификацию в виде сценариев пользователя.
Спецификация пишется по проекту с закрытым исходным кодом под название “HighFleet”
Пользовательский сценарий 1. Запуск сценария игры “Кампания”

Предусловие: приложение запущено.

Основной сценарий:
•	Нажатие на кнопку “Кампания”
•	Выбор ячейки сохранения
•	Указать имя ячейки сохранения 
•	Выбрать условие прохождения кампании (с прологом или без) 
•	Выбрать подходящие корабли из представленных, уложившись в определенную игрой сумму
•	Выбрать уровень сложности, используя переключатель в левом нижнем углу экрана
•	Нажать кнопку старт
•	Выбрать пройти ли  инструктаж

Постусловие: Игра полностью запущена и ждет действий от пользователя. 

Пользовательский сценарий 2. Запуск ракет дальнего действия.
Действующее лицо: Пользователь(Игрок).

Предусловие: приложение запущено, запущен сценарий игры “Кампания”.

Основной сценарий:
•	Выбираем тип ракет с помощью переключателя в правом верхнем углу экрана
•	Выбираем количество ракет, которые хотим запустить
•	Выбираем направление и цель ракеты
•	Нажимаем кнопку запуск в правом верхнем углу экрана

Постусловие: Возможный эффект на противника


Пользовательский сценарий 3. Дозаправка флота в городе.
Действующее лицо: Пользователь(Игрок).

Предусловие: приложение запущено, запущен сценарий игры “Кампания”.

Основной сценарий:
•	Долететь до ближайшего города, отмеченного на карте (Далее все действия будут происходить только в этом городе)
•	Произвести посадку корабля в доки
•	Зайти в магазин снабжения 
•	Выбрать количество топлива необходимое флоту
•	Заплатить за топливо (происходит автоматически, но это важная часть игры и должна быть указана)
•	Ожидание времени пока флот полностью за

Постусловие: Пополнение кол-ва топлива
